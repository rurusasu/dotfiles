# 10 - Gateway 設定

OpenClaw ゲートウェイサーバーに関する設定。
ネットワークバインド、認証、リロード戦略、管理 UI を制御する。

## 現在の設定値

### サーバー基本設定

| キー   | 値           | 説明                     |
| ------ | ------------ | ------------------------ |
| `port` | `18789`      | ゲートウェイの待受ポート |
| `mode` | `"local"`    | ローカル実行モード       |
| `bind` | `"loopback"` | localhost のみにバインド |

### reload

設定変更時のリロード動作を制御する。

| キー          | 値         | 選択肢                            | 説明                                       |
| ------------- | ---------- | --------------------------------- | ------------------------------------------ |
| `reload.mode` | `"hybrid"` | `hybrid`, `hot`, `restart`, `off` | 可能な場合はホットリロード、不可なら再起動 |

### auth

ゲートウェイ API の認証設定。

| キー         | 値                                                 | 説明                           |
| ------------ | -------------------------------------------------- | ------------------------------ |
| `auth.mode`  | `"token"`                                          | トークンベース認証             |
| `auth.token` | `op://Personal/openclaw/gateway token` (1Password) | 1Password から取得するトークン |

### tailscale

Tailscale VPN 連携設定。

| キー                    | 値      | 説明                                    |
| ----------------------- | ------- | --------------------------------------- |
| `tailscale.mode`        | `"off"` | Tailscale 連携を無効化                  |
| `tailscale.resetOnExit` | `false` | 終了時に Tailscale 状態をリセットしない |

### controlUi

Web ベースの管理画面設定。

| キー                | 値     | 説明             |
| ------------------- | ------ | ---------------- |
| `controlUi.enabled` | `true` | 管理 UI を有効化 |

## 設計判断

### loopback バインド

セキュリティ上、外部ネットワークからのアクセスを遮断するため `loopback` に限定する。
ローカル Docker 環境での運用を前提としており、外部公開の必要がない。

### hybrid リロード

`hybrid` モードは設定変更の種類に応じて最適なリロード戦略を自動選択する:

- ホットリロード可能な変更 (例: ログレベル、レート制限) → プロセス再起動なしで即時反映
- ホットリロード不可の変更 (例: ポート変更、認証方式) → 自動的にフル再起動

`hot` のみだと適用できない変更が発生し、`restart` のみだと不要なダウンタイムが生じる。

### token 認証 + 1Password

ゲートウェイ API をトークンで保護する。トークンは 1Password に格納し、chezmoi テンプレートで `op://` URI 経由で注入する。
ソースコードやリポジトリにシークレットが一切含まれない。

### Tailscale 無効化

ローカル Docker セットアップでは Tailscale によるリモートアクセスが不要なため無効化。
将来的にリモートアクセスが必要になった場合は `tailscale.mode: "authkey"` に変更して有効化できる。

### controlUi 有効化

Web ベースの管理画面を有効にし、ブラウザからセッション状態の確認や設定変更を行えるようにしている。
loopback バインドにより外部からはアクセスできないため、セキュリティリスクは低い。
