# サブエージェント設定 (`agents.defaults.subagents`)

OpenClaw のサブエージェント設定セクションについて記述する。

公式リファレンス: <https://docs.openclaw.ai/gateway/configuration>

## 現在の設定値

| パラメータ             | 値                           | 説明                               |
| ---------------------- | ---------------------------- | ---------------------------------- |
| `model`                | `openai-codex/gpt-5.3-codex`| サブエージェントが使用するモデル   |
| `maxConcurrent`        | `8`                          | 同時実行可能なサブエージェント数   |
| `maxSpawnDepth`        | `2`                          | サブエージェントの再帰生成上限     |
| `maxChildrenPerAgent`  | `5`                          | 1エージェントあたりの子エージェント上限 |
| `runTimeoutSeconds`    | `300` (5分)                  | サブエージェントのタイムアウト     |
| `archiveAfterMinutes`  | `30`                         | 完了セッションのアーカイブまでの時間 |

## 設定の根拠

### `model`: プライマリと同一モデル

プライマリエージェントと同じ `openai-codex/gpt-5.3-codex` を使用することで、応答品質とプロンプト互換性の一貫性を保つ。

### `maxConcurrent`: 8

親エージェントの `maxConcurrent=4` の2倍に設定。サブエージェントは個々のタスクが軽量なため、並列度を高めてスループットを確保する。

### `maxSpawnDepth`: 2

再帰的なサブエージェント生成を2階層に制限する。深いスポーン深度はコストとレイテンシを指数的に増大させるため、浅く抑える。

### `maxChildrenPerAgent`: 5

1エージェントあたりのファンアウトを5に制限し、リソース消費の爆発を防ぐ。

### `runTimeoutSeconds`: 300 (5分)

親エージェントのタイムアウト (600秒) より短い。サブエージェントはより小さなタスクを処理するため、短いタイムアウトで十分であり、ハングしたタスクの早期検出にも有効。

### `archiveAfterMinutes`: 30

完了したサブエージェントセッションを30分後にアーカイブし、メモリを解放する。

## 同時実行に関する注意事項

- 理論上の最大同時エージェント数: `maxConcurrent(4) * subagent.maxConcurrent(8) = 32`
- 実際の同時実行数はタスクの性質とリソース制約に依存する
- スポーン深度が深くなるほどコストとレイテンシが指数的に増加する

## モデル変更時の注意

プライマリモデルを GPT-5.4 などに変更する場合、サブエージェントのモデルも同時に変更すること。モデルの不一致はプロンプトの解釈差異やデバッグの困難さにつながる。
