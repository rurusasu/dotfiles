# 09 - Session 設定

OpenClaw のセッション管理に関する設定。
セッションのスコープ、リセット条件、メンテナンスポリシーを定義する。

## 現在の設定値

### dmScope

| キー      | 値                   | 説明                                                  |
| --------- | -------------------- | ----------------------------------------------------- |
| `dmScope` | `"per-channel-peer"` | チャンネル+ユーザーの組み合わせごとにセッションを分離 |

### reset

セッションリセットの動作を制御する。

| キー                | 値       | 選択肢                    | 説明                                      |
| ------------------- | -------- | ------------------------- | ----------------------------------------- |
| `reset.mode`        | `"idle"` | `idle`, `daily`, `manual` | 非アクティブ時に自動リセット              |
| `reset.idleMinutes` | `60`     | -                         | 60 分間操作がなければセッションをリセット |

### resetTriggers

| キー            | 値                   | 説明                           |
| --------------- | -------------------- | ------------------------------ |
| `resetTriggers` | `["/new", "/reset"]` | 手動リセットを発火するコマンド |

### maintenance

古いセッションの自動整理に関する設定。

| キー                     | 値       | 説明                                 |
| ------------------------ | -------- | ------------------------------------ |
| `maintenance.mode`       | `"warn"` | 古いセッションについて警告を出す     |
| `maintenance.pruneAfter` | `"7d"`   | 7 日以上経過したセッションを自動削除 |
| `maintenance.maxEntries` | `1000`   | セッションエントリの最大保持数       |

## 設計判断

### per-channel-peer スコープ

チャンネルとユーザーの組み合わせ単位でセッションを分離する。これにより:

- 同一ユーザーでもチャンネルが異なれば独立した会話コンテキストを維持
- チャンネル内で他ユーザーの会話が混入しない
- DM とグループチャットで文脈が混ざらない

### idle リセット (60 分)

`idle` モードを採用し、60 分の非アクティブでリセットする。

- `daily` だと日をまたぐ長時間作業中にセッションが切れるリスクがある
- `manual` だとユーザーがリセットを忘れ、古い文脈が蓄積する
- 60 分は「作業中の離席」と「会話終了」を区別する現実的な閾値

### resetTriggers: /new, /reset

明示的なリセット手段として 2 つのコマンドを用意。
`/new` は「新しい会話を始める」、`/reset` は「状態をクリアする」という異なるメンタルモデルに対応する。

### pruneAfter: 7d

7 日以上経過したセッションデータを自動削除し、ストレージの無制限な成長を防止する。
1 週間あれば過去の会話を参照する需要はほぼなくなるという運用上の判断。

### maxEntries: 1000

セッションエントリ数に上限を設け、メモリ・ストレージの際限ない増加を防ぐ。
1000 エントリは通常利用で数日分に相当し、十分な履歴保持量となる。
